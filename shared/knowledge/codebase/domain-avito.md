# Avito

Краткое описание возможностей Avaid (ветка dev, пакет `avito`).

## Бизнес-логика (из комментариев и кода backend-go)
- GetOrderLabelPDF запрашивает у Avito PDF с этикетками (наклейками) для одного заказа.
- Схема из публичного API: POST /order-management/1/orders/labels → taskId, затем
- GET /order-management/1/orders/labels/{taskID}/download (см. документацию «Управление заказами», generateLabels / downloadLabel).
- SyncProducts загружает все объявления из Avito API и синхронизирует с marketplace_products.
- SendPrices отправляет цены товаров с sync_prices=true в Avito API.
- SendStocks отправляет остатки товаров с sync_stocks=true в Avito API.
- rateLimiter — token-bucket предохранитель против 429 от Avito API.
- 1 токен = 1 запрос. Используется в Client перед каждым httpClient.Do,
- чтобы не сыпать API при массовой отправке цен/остатков (см. SendStocks/SendPrices).
- Wait блокируется, пока не появится 1 токен или ctx не отменится.
- nil-приёмник эквивалентен отсутствию ограничения (для тестов / опционального отключения).
- StockUpdate — один элемент пакетного обновления остатков в Avito stock-management.
- stockBatchMaxSize — Avito принимает массив stocks в одном запросе; держим небольшой батч,
- чтобы при бизнес-ошибках на отдельных позициях не терять весь пакет и не упираться в лимиты.
- UpdateItemPrice отправляет новую цену (в рублях) в Avito API.
- Эндпоинт не батчевый — путь содержит id товара, шлём по одному запросу с уважением Retry-After.
- UpdateItemStocksBatch отправляет массив остатков одним запросом в /stock-management/1/stocks.
- Возвращает map[item_id]сообщение_об_ошибке для позиций, которые Avito отверг бизнес-ошибкой
- (success=false с массивом errors). Транспортные ошибки и не-200 возвращаются обычным err.
- // Avito считает квоту по числу HTTP-запросов, поэтому батчинг — главный предохранитель против 429:
- раньше SendStocks делал 1 запрос на каждый offer_id, теперь — 1 запрос на батч.
- authedJSONRaw отвечает только за транспорт (Wait → Do → retry на 429 с Retry-After).
- Возвращает сырое тело успешного ответа; не-2xx (кроме 429) и сетевые ошибки превращаются в err.
- authedJSON — обёртка над authedJSONRaw для одиночных запросов (UpdateItemPrice).
- Дополнительно парсит бизнес-ошибки из тела 2xx и превращает их в err.
