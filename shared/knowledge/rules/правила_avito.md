# Специальные правила AI-агента: Avito

## Общий принцип
Avito работает по стандартной логике: товар на маркетплейсе связывается с товаром в МойСклад, после чего приложение управляет остатками и ценой. Приложение НЕ создаёт новые объявления и НЕ переносит товары между маркетплейсами.

---

## 1. Связка товара — по ID объявления

**Суть:** Для Avito основной идентификатор — ID объявления. Это числовое значение прямо под названием объявления.

**Как объяснять:** Для связки товара нужно ID объявления — номер под названием объявления в Avito.

**Нельзя говорить:** что связка выполняется по внутренним техническим данным; нельзя путать ID с артикулом, SKU или названием.

**Когда передавать специалисту:** если клиент не может найти ID или связка по ID не проходит.

---

## 2. Ручная связка и автопривязка

**Суть:** Товары можно связать вручную (по одному) или через автопривязку. Для автопривязки нужно заранее создать дополнительное поле в МойСклад.

**Что уточнить:** создано ли поле в МойСклад; правильно ли оно названо; выбран ли параметр Avito при автопривязке.

**Нельзя:** обещать что автопривязка свяжет всё без проверки; запускать автопривязку сразу по всем параметрам без контроля.

---

## 3. Дополнительное поле «Avito» в МойСклад

**Требования к полю:**
- Название: **Avito** (с большой буквы, на английском, без лишних пробелов)
- Тип: **Строка**
- Обязательное: **нет**

**Как создать:**
1. Открыть карточку товара в МойСклад
2. Нажать на шестерёнку → перейти в самый низ настроек → добавить дополнительное поле
3. Назвать поле «Avito», тип «Строка»
4. Вставить ID объявления из Avito в это поле
5. Запустить автопривязку по параметру «Avito / Avito»

**Нельзя:** называть поле по-русски «Авито»; указывать числовой тип вместо строки.

**Когда передавать специалисту:** если поле создано, но оно не появляется в товарах или автопривязка его не видит.

---

## 4. После связки — управление остатками и ценой

**Суть:** После правильной связки приложение передаёт остаток и цену из МойСклад в объявление Avito. Создаёт отгрузки и возвраты.

**Нельзя говорить:** что приложение само создаёт новые объявления; что есть двусторонняя выгрузка между маркетплейсами.

**Когда передавать специалисту:** если связка есть, но цена или остаток по конкретному объявлению не обновляется.

---

## 5. Avito доступен не на всех тарифах

**Avito входит в тариф Премиальный** (без доплаты).

**На остальных тарифах** — Avito подключается отдельно за **3 000 рублей**.

**Что уточнить:** текущий тариф клиента.

**Как объяснять:** Avito входит в Премиальный тариф. На других тарифах его можно подключить отдельно за 3 000 рублей.

**Нельзя:** обещать скидку или бесплатное подключение; подключать услугу самостоятельно.

**Когда передавать специалисту:** если клиент просит скидку, индивидуальные условия или подключение.

---

## 6. Интеграция только через API на бизнес-тарифах Avito

**Суть:** Интеграция с Avito работает только через API и доступна только для **бизнес-аккаунтов** Avito. На аккаунтах физических лиц и тарифах без API — интеграция невозможна.

**Что уточнить:** тип аккаунта Avito; тариф Avito; есть ли доступ к API-ключам.

**Нельзя:** обещать подключение на аккаунте физлица; говорить что проблему можно обойти внутри приложения.

---

## 7. API-ключи Avito — только в браузере, не в мобильном

**Суть:** API-ключи отображаются только в полной версии Avito через браузер. В мобильном приложении Avito ключи не показываются.

**Что уточнить:** с какого устройства клиент открывает Avito.

**Как объяснять:** API-ключи доступны только в полной версии сайта через браузер. В мобильной версии они не отображаются.

---

## 8. Разнесение денежных средств по Avito — временно недоступно

**Суть:** Функция разнесения финансов по Avito временно не работает, будет добавлена позже.

**Нельзя:** называть точную дату добавления если она не утверждена; говорить что функция уже работает.

**Когда передавать специалисту:** если клиенту нужна индивидуальная доработка или точные сроки.

---

## 9. Двусторонней выгрузки нет

**Суть:** Нельзя перенести товары с одного маркетплейса на Avito или создать новые объявления через приложение. Приложение управляет только уже существующими связанными объявлениями.

**Нельзя обещать:** создание объявлений на Avito через приложение; перенос карточек между маркетплейсами.

---

## 10. Автопродление объявлений нужно включить в Avito вручную

**Суть:** Если объявление снялось с публикации — приложение не может поднять его повторно. Нужно включить **автопродление** в настройках объявления в самом Avito.

**Что уточнить:** ID объявления; опубликовано ли оно сейчас; включено ли автопродление в настройках объявления Avito.

**Как объяснять:** Чтобы объявление не снималось с публикации, в Avito нужно включить автопродление. Если объявление уже снялось — приложение не сможет поднять его самостоятельно.

**Нельзя обещать:** что приложение само восстановит снятое объявление.

**Когда передавать специалисту:** если автопродление включено, но объявление всё равно снимается.
