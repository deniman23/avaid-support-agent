# Специальные правила AI-агента по Ozon

Общий принцип: AI не раскрывает клиенту внутреннюю техническую реализацию. Объясняет ограничения Ozon простыми словами, собирает конкретный пример и при необходимости передаёт специалисту.

## Технические особенности Ozon

1. Модель магазина (FBO/FBS) выбирается при создании и дальше не меняется
2. Telegram-чат доступен не на всех тарифах
3. Для первичной настройки Telegram-чат нужно сделать публичным
4. Один Telegram-чат можно использовать для нескольких магазинов
5. Единый PDF по нескольким магазинам требует одинакового времени отправки
6. Telegram-уведомления в России могут не доходить при выключенном VPN
7. В связке товаров Ozon можно связывать обычные товары и комплекты
8. Удаление из акций Ozon происходит в момент начала акции
9. Защита по минимальной цене отдельно не требуется при удалении из акций
10. Документация по отгрузкам для расчёта выплат не нужна
11. В заказах есть «Цена продавца» и «Выплата продавцу»
12. Столбец «Выплата продавцу» доступен не на всех тарифах
13. Для возвратов Ozon нужен специальный barcode
14. PDF с barcodes возвратов доступен на всех тарифах
15. Кнопка «Barcodes» формирует единый PDF по текущим отгрузкам
16. На базовых тарифах PDF нужно формировать вручную
17. Раздел «Заказы» показывает единую ленту заказов и статусов
18. Ozon FBO не привязан к конкретному складу внутри решения
19. Физическая отгрузка Ozon FBO создаётся вне решения
20. Видимость остатков Ozon FBO по конкретным складам пока не реализована
21. Дополнительные FBO-отгрузки через решение сформировать нельзя
22. Защита от акций не защищает от подписок Ozon
23. Резервация товара происходит с интервалом около 3 секунд
24. При включённом управлении остатками решение не уводит товар в минус
25. Для самостоятельной продажи в минус нужно отключить управление остатками на магазине
26. Не все формы отгрузки Ozon доступны на всех тарифах
27. Есть ограничения по количеству товаров в зависимости от тарифа

## Карточка для передачи специалисту

- Маркетплейс: Ozon
- Магазин: название в решении
- Модель работы: FBS / FBO
- Товар: название, артикул или SKU
- Что клиент ожидал
- Что отображается фактически
- Когда заметил проблему
- Включены ли нужные рубильники
- Тариф клиента (если вопрос связан с доступностью функции)
- Скриншот без токенов и личных данных

**Важно:** AI не обещает исправление, скидку, изменение тарифа или наличие функции пока это не подтверждено специалистом.
