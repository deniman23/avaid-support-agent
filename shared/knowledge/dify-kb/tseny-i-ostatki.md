# Цены и остатки

## Что это и зачем

Avaid берёт остатки и цены из МойСклад и автоматически отправляет их на маркетплейсы. Без этого остатки на Ozon/WB/Яндексе устаревают, и покупатели заказывают товар, которого нет в наличии.

---

## Как работает синхронизация остатков

Схема одинакова для всех маркетплейсов (Ozon, WB, Яндекс, Avito):

1. Avaid читает остатки из МойСклад по выбранным складам
2. Агрегирует количество по режиму (сумма или один склад)
3. Отправляет итоговое количество на маркетплейс

### Режимы агрегации остатков

В настройках магазина → вкладка **«Склады»** есть выбор режима:

**Сумма складов** (по умолчанию) — складывает остатки по всем выбранным складам МС. Пример: склад А — 5 шт., склад Б — 3 шт. → на маркетплейс уходит 8.

**Один склад** — берёт остаток только с одного выбранного склада МС. Используют, когда у разных магазинов свои склады и не нужно суммировать.

**Отключено** — остатки не синхронизируются совсем (можно отключить только отправку остатков, не трогая цены).

### Два вида складов в настройках

Для каждого магазина настраивается два списка складов (вкладка **«Склады»**):

**Склады МойСклад** — откуда Avaid читает остатки. Это склады в вашей системе учёта. При режиме «Один склад» отображается выпадающий список **«Выберите склад»**, при режиме «Сумма складов» — мультивыбор **«Выберите склады для суммирования»**.

**Склады маркетплейса** — куда Avaid отправляет остатки. Для каждого маркетплейса свои:
- WB: поле **«Выберите склады Wildberries»** — склады Wildberries
- Ozon: поле **«Выберите склады Ozon»** — FBO-склады или FBS-адреса

Оба списка должны быть заполнены, иначе синхронизация остатков не запустится.

---

## Как работает синхронизация цен

### Откуда берётся цена для отправки

Цена определяется в таком приоритете:

1. **Формула** в карточке товара + рассчитанная цена — если задана формула и есть рассчитанное значение, используется оно
2. **Уровень цены МойСклад** — если у товара или магазина указан уровень цен МС, берётся цена из этого уровня
3. **Цена с маркетплейса** — если ничего из выше не задано, остаётся цена, которую Avaid последний раз получил с маркетплейса

### Особенности по маркетплейсам

**Wildberries** — для отправки цен нужен отдельный токен (в настройках магазина вкладка **«Цены»** → поле **«Токен для цен WB»**). Без него цены не обновляются. Скидка ограничена 0–99%.

**Ozon** — цена и скидка отправляются через API обновления цен.

**Яндекс.Маркет** — в настройках магазина вкладка **«Цены»** есть поле **«Режим синхронизации цен»**: обновлять цены в кабинете Яндекса или в магазине (или оба).

## Управление синхронизацией

### Включение/выключение для всего магазина

В настройках магазина (вкладки **«Склады»** и **«Цены»**):
- Переключатель **«Синхронизация остатков»** — включить/выключить отправку остатков для всего магазина
- Переключатель **«Синхронизация цен»** — включить/выключить отправку цен для всего магазина

Если выключено — Avaid не трогает остатки/цены на маркетплейсе даже при синхронизации.

### Включение/выключение для отдельного товара

В разделе **«Цены и остатки»** для каждого товара в таблице есть колонки:
- **«Синхронизация остатков»** — отправлять ли остатки по этому товару
- **«Синхронизация цен»** — отправлять ли цены по этому товару
- **«Удалять из акций»** — исключить товар из акций (цены не трогаются при акционных обновлениях)

## Почему остатки не обновляются

### 1. Товар не привязан к МойСклад
Avaid не знает, откуда брать остатки. Нужна автопривязка или ручная привязка (см. документ «Автопривязка»).

### 2. Не выбраны склады
Проверьте в настройках магазина (вкладка **«Склады»**): заполнены ли оба поля — склады МойСклад и склады маркетплейса. Если хотя бы одно пустое — остатки не пойдут.

### 3. Синхронизация остатков выключена
- Переключатель **«Синхронизация остатков»** выключен для магазина, или
- Переключатель **«Синхронизация остатков»** выключен у конкретного товара в таблице

### 4. МойСклад вернул пустой ответ
Avaid не обнуляет остатки при временно пустом ответе МС (защита от ложного обнуления). Если остатки долго не обновляются — проверьте подключение МойСклад.

### 5. У товара нет идентификатора WB (только WB)
Для WB нужен внутренний ID размера WB. Если он не заполнен, товар пропускается. Помогает синхронизация товаров WB.

---

## Почему цены не обновляются

### 1. Нет токена цен (только WB)
Для WB нужен отдельный токен в поле **«Токен для цен WB»** (настройки магазина → вкладка **«Цены»**).

### 2. Синхронизация цен выключена
Переключатель **«Синхронизация цен»** выключен для магазина или для конкретного товара.

### 3. Товар исключён из акций
Включён переключатель **«Удалять из акций»** — товар не участвует в ценовых обновлениях.

### 4. Цена равна нулю
Avaid не отправляет цены ≤ 0.

## Часто задаваемые вопросы

**В: Остатки обновились в МойСклад, но на маркетплейсе всё ещё старые. Сколько ждать?**
О: Синхронизация происходит при следующей отправке по расписанию или вручную. Если прошло больше часа — проверьте настройки складов и привязку товаров.

**В: Настроил два склада МС. Хочу, чтобы на маркетплейс уходила сумма с обоих.**
О: Это режим «Сумма складов» — он по умолчанию. Убедитесь, что оба склада добавлены в поле «Выберите склады для суммирования» в настройках магазина.

**В: Хочу для разных магазинов разные склады МС.**
О: Каждый магазин в Avaid настраивается отдельно — у каждого свой список складов МС.

**В: Для WB остатки не идут, хотя всё настроено.**
О: Проверьте, что заполнено поле **«Выберите склады Wildberries»** — это отдельное поле от складов МС. Также убедитесь, что у товаров синхронизированы данные с WB (нужен внутренний ID размера).

**В: Хочу, чтобы цена на маркетплейсе была с наценкой от цены МС.**
О: Настройте формулу цены в карточке товара. После этого Avaid будет рассчитывать цену по формуле и отправлять её.

**В: Цена на Ozon не меняется, хотя я изменил цену в МойСклад.**
О: Проверьте, что у товара указан **«Уровень цены»** МС или настроена формула. Если нет ни формулы, ни уровня — Avaid оставит цену с маркетплейса.
