# МойСклад — интеграция

## Что это и зачем

МойСклад — система учёта, с которой Avaid синхронизирует данные в обе стороны: забирает товары, остатки, склады, цены — и создаёт документы по заказам (заказы покупателя, отгрузки, возвраты).

---

## Подключение МойСклад

Для работы Avaid использует токен доступа к МойСклад. Токен выдаётся при установке приложения Avaid в МойСклад или при подключении через API.

**Если подключение не работает:** В МойСклад могут быть отозваны права приложения или истёк ключ. Симптом — синхронизация перестаёт работать, появляется ошибка «Неправильный или просроченный ключ доступа». Нужно переподключить приложение в МойСклад.

---

## Что синхронизируется из МойСклад

При полной синхронизации данных МС Avaid загружает:

- **Товары** — все товары и варианты, не в архиве. Включает артикул, код, внешний код, наименование, цены продажи, штрихкоды, атрибуты.
- **Остатки** — количество по каждому складу для каждого товара.
- **Склады** — список складов пользователя в МС.
- **Организации** — юридические лица.
- **Контрагенты** — покупатели и поставщики.
- **Статусы заказов покупателя** — кастомные статусы из МС для маппинга.
- **Проекты, договоры, каналы продаж** — для заполнения документов.
- **Составы комплектов** — компоненты для расчёта остатков.

### Особенность синхронизации товаров

Товары запрашиваются страницами по 100 с expand цен. Это ограничение API МойСклад. Все товары без архивного флага.

### Защита от ложного обнуления остатков

Если МойСклад вернул пустой ответ по остаткам, Avaid **не обнуляет** ранее загруженные остатки. Это защита от временных сбоев API МС. Если остатки долго не обновляются — проверьте подключение.

---

## Комплекты (bundle)

Комплект в МС — это товар, состоящий из нескольких компонентов. Avaid умеет с ними работать:

1. Загружает состав комплекта из МС
2. Рассчитывает остаток комплекта — это минимум по остаткам всех компонентов
3. Записывает рассчитанный остаток в базу для отправки на маркетплейс

**Пример:** Комплект из товара А (5 шт. на складе) и товара Б (3 шт. на складе) → остаток комплекта = 3 шт.

---

## Ценовые уровни МойСклад

В МС можно задать несколько уровней цен (оптовая, розничная и т.д.). Avaid поддерживает это:

- Для каждого магазина можно указать **«Уровень цены по умолчанию»** в настройках магазина → вкладка **«Цены»**.
- Для конкретного товара можно задать **«Уровень цены»** прямо в карточке товара — он перекрывает настройку магазина.
- Если уровень не задан — используется основная цена продажи.
- Для вариантов товара: если у варианта нет цены по уровню — ищется цена у родительского товара.

## Документы в МойСклад по заказам

Avaid создаёт три вида документов:

**Заказ покупателя** — создаётся при поступлении заказа с маркетплейса. Содержит список товаров, покупателя, сумму.

**Отгрузка** — создаётся, когда статус заказа требует отгрузки (товар передан в доставку). Требует наличия «Заказа покупателя».

**Возврат покупателя** — создаётся при возврате заказа. Привязывается к «Заказу покупателя».

### Настройки создания документов

В настройках магазина → вкладка **«Заказы»**:

- **«Создавать Заказы покупателя»** — без этой настройки никакие документы не создаются.
- **«Создавать Отгрузки»** — создавать ли документ отгрузки при отправке товара.
- **«Создавать Возвраты покупателя»** — создавать ли документ возврата в МС (в настройках возвратов).

---

## Реквизиты документов МойСклад

В настройках магазина → вкладка **«Заказы»** задаются реквизиты, которые Avaid проставляет в документы:

- **«Организация»** — от чьего имени создаётся заказ/отгрузка
- **«Контрагент»** — покупатель в документе
- **«Склад для отгрузок/заказов»** — откуда производится отгрузка
- **«Проект»** — проект в МС (необязательно)
- **«Договор»** — договор с покупателем (необязательно)
- **«Канал продаж»** — источник продажи (необязательно)
- **«Цены в документах»** — какой уровень цен проставлять в документах

Для возвратов (вкладка **«Возвраты»**):
- **«Склад при создании возврата»** — куда приходит товар при возврате
- **«Склад при завершении возврата»** — финальный склад после завершения возврата

---

## Ошибки и их значение

**«Неправильный или просроченный ключ доступа»** — токен МС недействителен. Нужно переподключить приложение в МС. Синхронизация автоматически не повторяется при этой ошибке.

**Превышение лимита запросов** — слишком много запросов к API МС. Avaid автоматически делает паузу и повторяет.

**Бизнес-ошибка** — заказ/возврат не найден, магазин неактивен, подписка истекла. Задача помечается как выполненная — повтор не поможет.

---

## Часто задаваемые вопросы

**В: Товары в МС обновились, но в Avaid старые данные.**
О: Нужна синхронизация товаров МС. Она запускается автоматически по расписанию. Если срочно — запустите вручную через интерфейс.

**В: Остатки в МС есть, а в Avaid показывает 0.**
О: Проверьте, что склады МС добавлены в настройки магазина (вкладка **«Склады»** → поле с выбором складов МойСклад). Если склады не выбраны — Avaid не знает, с какого склада брать остатки.

**В: Создал комплект в МС, но остатки по нему не считаются.**
О: Нужна синхронизация состава комплектов. После синхронизации Avaid пересчитает остатки комплекта по компонентам.

**В: В МС удалил товар, а в Avaid он ещё есть.**
О: После синхронизации товаров Avaid удалит устаревшие записи. Архивные товары фильтруются при синхронизации.

**В: Почему цена из МС не попадает на маркетплейс?**
О: Возможные причины: не настроен **«Уровень цены по умолчанию»** для магазина или **«Уровень цены»** для товара, или у товара в МС не заполнена цена продажи. Также проверьте, включена ли **«Синхронизация цен»** в настройках магазина.

**В: Что значит ошибка «Неправильный или просроченный ключ доступа»?**
О: Токен подключения МойСклад истёк или был отозван. Зайдите в МойСклад → Приложения → найдите Avaid → переподключите. После этого синхронизация возобновится.
