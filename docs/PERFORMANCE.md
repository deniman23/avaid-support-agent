# Скорость и язык ответов Avaid Support

## Почему медленно

1. **Ollama локально** (`qwen2.5:7b`) — на CPU один проход LLM занимает десятки секунд.
2. **Agent mode** — до 3 итераций: LLM → tool → LLM → …
3. **3 базы знаний** — поиск по Rules + Codebase + Schema перед ответом.
4. **11 tools** — большая схема function calling, модель дольше «думает».

## Почему «размышления» на английском

Dify показывает поле **agent_thought** — это текст, который модель сгенерировала **до** вызова tool. У Qwen часто это английский план («I will check…»). Это не отдельная настройка Dify, а поведение модели.

В промпте запрещены длинные рассуждения; модель должна сразу вызывать tools.

## Как ускорить

| Действие | Эффект |
|----------|--------|
| GPU для Ollama | Сильно быстрее |
| Модель `qwen2.5:3b` | Быстрее 7b, хуже качество |
| Меньше итераций (сейчас 3) | Меньше кругов LLM |
| Отключить лишние datasets в Studio | Быстрее RAG |
| `num_predict` 600 (в конфиге) | Короче ответы |

## Studio

**Avaid Support → Model** — при наличии GPU можно попробовать `qwen2.5:14b` (качество ↑, скорость ↓ на CPU).

После изменений: `./scripts/dify-fix-agent.sh` и новый диалог (F5).
